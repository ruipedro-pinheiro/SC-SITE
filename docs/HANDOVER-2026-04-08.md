## 7. The CryXML blocker — exact fix for tomorrow

**Problem**: scunpacked Loader's ShipLoader phase crashes on the first ship it
tries to parse, because 131 out of 159 vehicle XMLs under
`Data/Scripts/Entities/Vehicles/Implementations/Xml/` are CryXML binary
(magic bytes `CryXmlB\0`), and scunpacked has zero CryXML support — it does
`File.ReadAllText` + `XmlDocument.LoadXml(xml)`, throwing:
```
Unhandled exception. System.Xml.XmlException: Data at the root level is invalid. Line 1, position 1.
   at Loader.VehicleParser.LoadVehicleXml(String vehiclePath) in ...VehicleParser.cs:line 56
```

**Fix**: pre-decode the CryXML binaries into plain XML under a shadow dir, then
repoint the loader-input symlink. Cache is saved, so Loader re-run skips the
13.4 min indexing.

**Resume script** (ready to run on VM tomorrow):

```bash
ssh admin727366@100.70.4.2
source ~/sc-extract/scdt-venv/bin/activate
export PATH=$HOME/.dotnet:$PATH

# Kill any leftover Loader
pkill -f "Loader.dll" 2>/dev/null
pkill -f "scunpacked/Loader/bin/Release" 2>/dev/null

# Decode all CryXML under Data/Scripts (scdatatools cryxml expects a file object, not a path!)
python3 <<'PYEOF'
from pathlib import Path
from scdatatools.engine.cryxml import etree_from_cryxml_file

SRC = Path("/media/admin727366/games/SC DATA/live-4.7.176.58286-11592622/raw-text-xml/Data/Scripts")
DST = Path("/home/admin727366/sc-extract/scripts-decoded/Data/Scripts")

decoded = skipped = failed = 0
fails = []
for src in SRC.rglob("*.xml"):
    if not src.is_file():
        continue
    rel = src.relative_to(SRC)
    dst = DST / rel
    dst.parent.mkdir(parents=True, exist_ok=True)

    with open(src, "rb") as f:
        magic = f.read(7)

    if magic != b"CryXmlB":
        # plain XML — symlink if not already
        if not (dst.is_symlink() or dst.exists()):
            try:
                dst.symlink_to(src)
                skipped += 1
            except Exception:
                failed += 1
        continue

    # CryXML binary → decode
    try:
        with open(src, "rb") as f:
            tree = etree_from_cryxml_file(f)
        if dst.is_symlink() or dst.exists():
            dst.unlink()
        tree.write(str(dst), encoding="utf-8", xml_declaration=True)
        decoded += 1
    except Exception as e:
        failed += 1
        if len(fails) < 10:
            fails.append(f"{rel}: {type(e).__name__}: {e}")

print(f"decoded={decoded} non_cryxml_symlinks={skipped} failed={failed}")
for f in fails:
    print("FAIL", f)
PYEOF

# Verify first decoded file is plain text
head -c 200 ~/sc-extract/scripts-decoded/Data/Scripts/Entities/Vehicles/Implementations/Xml/CRUS_Starlifter.xml
# Expected: <?xml version="1.0" encoding="utf-8"?><Vehicle... (plain text, NOT "CryXmlB")

# Loader-input symlink should already point to the shadow (set earlier this session)
ls -la ~/sc-extract/loader-input/Data/Scripts

# Re-run Loader (should skip 13.4 min cache rebuild since cache is saved in loader-input/)
cd ~/sc-extract/scunpacked/Loader
nohup dotnet bin/Release/net8.0/Loader.dll \
  -input=$HOME/sc-extract/loader-input \
  -output=$HOME/sc-extract/json-out \
  -noitems -noshops -nomap -nomissions \
  > /tmp/loader-run4.log 2>&1 &
echo "Loader PID: $!"

# Wait, then verify
sleep 30
tail -30 /tmp/loader-run4.log
find ~/sc-extract/json-out/ships -type f | wc -l
cat ~/sc-extract/json-out/ships/anvl_carrack.json | python3 -m json.tool | head -80
```
